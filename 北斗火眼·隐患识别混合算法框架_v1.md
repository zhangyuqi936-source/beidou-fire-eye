# 北斗火眼·隐患识别混合算法框架 v1（BDHY-Hybrid Hazard Recognition Agent v1）

> **版本标识**：`agentVersion: 1` · `algorithmVersion: v1` · `rankMethod: hybrid_risk_v1`  
> **设计日期**：2026-06-27 · **状态**：框架设计阶段  
> **底座模型**：DeepSeek Vision（多模态识别）+ mxbai-embed-large（法规语义匹配）  
> **文献支撑**：GB 35181-2025 · GB 50016-2014 · GB 55037-2022 · GB 55036-2022 · 三维风险矩阵（MDPI Fire 2024）· DetectNet_v2（Frontiers 2025）· 监管RAG（MDPI Electronics 2025）· 多模态施工安全（AEI 2025）

---

## 1. 设计目标

北斗火眼的隐患识别不是「拍张照让 DeepSeek 看一下然后随便说几个问题」，而是：

1. **LLM 只做擅长的事**：多模态视觉理解、自然语言描述隐患、生成可行动整改建议、解释法规依据。
2. **结构化知识库 + 本地规则负责速度与稳定性**：隐患分类树索引、法规向量库语义检索、风险矩阵量化、多级筛选。
3. **可观测、可降级**：前端展示 `chain` 识别链路；`meta` 标明是否命中 LLM / 缓存；API Key 缺失或超时仍返回规则检查结果。
4. **法规可追溯**：每条隐患标注对应的 GB 标准条款编号，支持一键查看原文。
5. **闭环进化**：人工复核反馈 → 分类权重调整 → 向量库增量更新。

## 2. 总体流程

```text
用户拍照 / 上传图片 + 可选场所类型标签
        │
        ▼
[Preprocess] 图像预处理 — 分辨率归一化 (1280×720)、EXIF 方向校正、光照均衡
        │
        ▼
[Scene] sceneClassify() — 规则+LLM 识别场所类型（8大类·32子类）
        │                      失败则规则兜底
        ▼
[Detect] 三层视觉识别管线
        │   ├─ Layer 1: 消防设施状态检测（灭火器/消火栓/报警器/应急灯/疏散标志）
        │   ├─ Layer 2: 通道与环境检测（疏散通道堵塞/防火门状态/安全出口/杂物堆积）
        │   └─ Layer 3: 危险源检测（电气隐患/易燃物/明火源/烟雾）
        ▼
[Classify] 隐患分类映射 — 视觉发现 → GB 35181-2025 分类树节点
        │
        ▼
[Risk] 三维风险量化 — 严重性 × 可能性 × 可检测性 → 风险等级 (I/II/III/IV)
        │
        ▼
[Regulation] 法规语义匹配 — mxbai-embed-large 向量检索 + 规则关键词兜底
        │
        ▼
[Generate] 隐患报告生成 — LLM 结构化输出（隐患描述 + 风险等级 + 法规依据 + 整改建议）
        │
        ▼
[Side effects] logInspection → inspection_records
              setCachedResult → inspection_cache（5min TTL）
        │
        ▼
返回 items + scene + chain + meta
```

## 3. 模块与职责

| 模块 | 职责 | 核心依赖 |
|------|------|----------|
| **SceneClassifier** | 场所类型识别（8大类32子类） | 规则词典 + DeepSeek Vision |
| **HazardDetector** | 三层视觉识别管线编排 | DeepSeek Vision + 结构化Prompt |
| **HazardTaxonomy** | 隐患分类树索引与映射 | GB 35181-2025 分类体系 |
| **RiskQuantifier** | 三维风险矩阵量化打分 | 严重性×可能性×可检测性 |
| **RegulationMatcher** | 法规语义检索与条款匹配 | mxbai-embed-large + 向量库 |
| **ReportGenerator** | 结构化隐患报告输出 | DeepSeek LLM |
| **DegradationEngine** | 降级策略管理与离线模式 | 规则引擎 |
| **FeedbackLoop** | 人工复核反馈→模型微调闭环 | inspection_records 聚合 |

### 3.1 模块间的解耦设计

每个模块独立可替换：
- 替换视觉模型：只需遵循 `HazardDetector` 的输入输出接口
- 替换法规库：只需重建 `RegulationMatcher` 的向量索引
- 替换风险模型：只需更新 `RiskQuantifier` 的权重矩阵

### 4.3 视觉可检测性矩阵

并非所有隐患类别都能从单张照片中直接检测。以下矩阵定义了每个隐患类别的**视觉可检测性**和**建议的检测方式**：

| 检测方式 | 编码 | 说明 |
|----------|------|------|
| `visual` | V | 单张照片可直接判断（如灭火器缺失、通道堵塞） |
| `visual-context` | VC | 需要场景上下文辅助判断（如常闭防火门未关闭需要识别门的类型） |
| `infer` | I | 需要 LLM 推理推断（如"线路老化"需要综合电线外观+环境推断） |
| `non-visual` | N | 照片无法检测（如消防演练记录缺失），仅规则提示 |

## 5. 视觉识别层（Visual Recognition Layer）

### 5.1 底座模型选型论证

| 候选模型 | 优势 | 劣势 | 结论 |
|----------|------|------|------|
| **DeepSeek Vision (VL2)** | 已在北斗火眼中使用，MoE架构高效，中文多模态理解强 | 非专用安全检测模型 | **首选** — 性价比最高 |
| YOLOv8 + 专用分类器 | 检测速度快，可离线部署 | 需要大量标注数据训练，不支持自然语言输出 | 备选 — 用于离线降级场景的特定目标检测 |
| DetectNet_v2 + ResNet-18 | 火灾/烟雾检测95.6%/92%精度 | 仅检测火焰烟雾，不支持通用隐患 | 辅助 — 作为火灾预警的快速通道 |

**决策**：主链路使用 DeepSeek Vision VL2，火焰/烟雾快速预警通道使用轻量检测模型（DetectNet_v2 或 YOLOv8s）。

### 5.3 DeepSeek Vision Prompt 工程

**核心原则**（基于文献中多模态LMM施工安全巡检的LoRA微调经验）：
1. **结构化输出 Schema 先行**：先定义 JSON Schema，再让模型填空，而非让模型自由发挥
2. **负向约束**：明确告知哪些不是隐患（如"装饰性灯具不等于应急灯"）
3. **场所感知**：传入场所类型标签作为上下文，缩小检测范围
4. **分级置信度**：要求模型为每个发现标注 `confidence: high|medium|low`

#### System Prompt（系统层）

```
你是一个消防隐患智能识别系统。你的任务是从照片中识别消防安全隐患。

核心原则：
1. 只报告你确实看到的内容，不要推测看不到的
2. 每个发现必须标注置信度：high（清晰可见）/ medium（基本可判断）/ low（不确定，需人工确认）
3. 对于不确定的发现，标注为 low 置信度并在备注中说明
4. 不要将装饰性物品误判为消防设施
5. 严格按照 JSON Schema 输出，不要添加额外字段
```

#### 分层检测 Prompt（以 Layer 1 为例）

**Layer 1 — 消防设施状态检测 Prompt：**

```
请从照片中检测以下消防设施的状态：

检测清单：
- 灭火器：是否存在？数量是否充足？压力表是否在绿色区域？是否被遮挡？
- 消火栓：箱体是否完整？水带水枪是否齐全？是否被遮挡/封堵？
- 火灾报警器：探测器是否存在？指示灯是否正常？是否被遮挡/涂覆？
- 自动喷淋：喷淋头是否存在？是否被遮挡/涂覆？管网是否可见？
- 应急照明灯：是否存在？外观是否完好？
- 疏散指示标志：是否存在？是否亮起？方向是否正确？

对每个发现的设施，必须标注：
1. status: normal|missing|damaged|blocked|expired|unknown
2. confidence: high|medium|low
3. location_hint: 设施在画面中的位置描述
4. note: 补充说明（如压力表读数、具体损坏情况）
```

## 6. 法规匹配层（Regulation Matching Layer）

### 6.1 设计理念

不同于简单的关键词匹配（查"灭火器"→返回灭火器条款），法规匹配层应该理解 **语义场景**。例如：
- 照片中检测到「疏散走道被纸箱堵塞 + 场所为商场」→ 应匹配 GB 50016 §5.5（疏散走道净宽度要求）+ 《消防法》第28条（不得占用疏散通道）
- 而非简单返回所有包含"疏散"关键词的条款

**核心方案**：**语义向量检索（主）+ 规则关键词检索（兜底）+ LLM 条款精排（可选）**

### 6.2 法规知识库构建

| 步骤 | 方法 | 说明 |
|------|------|------|
| **法规采集** | 手动整理 + 定期更新 | GB系列标准全文、消防法及配套规章、地方消防条例 |
| **文本分割** | 按条款/段落边界分割 | 不按固定字符数切分，保持条款完整性（文献支持：基于章节的语义分割） |
| **元数据标注** | 标准编号 + 条款号 + 场所类型标签 | 每条 chunk 标注适用的场所类型、隐患类别 |
| **向量化** | mxbai-embed-large (1024维) | 文献验证：在监管文本语义匹配中综合性能最优 |
| **存储** | 向量数据库 + 全文索引双通道 | 向量检索负责语义匹配，全文索引负责关键词精确查找 |

### 6.3 法规匹配流程

```text
隐患识别结果（分类编码 + 场所类型 + 自然语言描述）
        │
        ▼
┌──────────────────────────────────────┐
│  查询向量构建                          │
│  输入: "{场所类型} {隐患类别}          │
│         {隐患描述}"                    │
```

## 7. 风险量化层（Risk Quantification Layer）

### 7.1 设计基础：三维风险矩阵

参考 MDPI Fire (2024) 的三维风险矩阵方法论，将其从实验室安全场景适配到消防安全巡检场景。**关键调整**：将第三维度从「预防性（Preventive）」改为「可检测性（Detectability）」，因为北斗火眼的使用场景是"通过照片发现隐患"而非"评估单位的预防能力"。

```
综合风险 = 严重性(Severity) × 可能性(Likelihood) × 可检测性(Detectability)
         = S × L × D
```

三维各4个等级，共 **4×4×4 = 64** 种组合。

### 7.2 严重性（Severity）评分体系

| 等级 | 分值 | 标准 | 判定依据 |
|------|------|------|----------|
| S1 | 1 | 轻微 | 不影响人员安全疏散，无直接火灾风险（如应急灯罩破损但不影响功能） |
| S2 | 2 | 一般 | 可能影响疏散或初期灭火，但不会直接导致火灾扩大（如个别灭火器过期） |
| S3 | 3 | 严重 | 严重影响疏散或灭火能力，可能导致火势蔓延（如疏散通道被严重堵塞） |
| S4 | 4 | 极严重 | 直接威胁生命安全或必然导致重大火灾（如火灾报警系统完全瘫痪） |

**判定辅助**：严重性评分参考 GB 35181-2025 的直接判定要素——触及其30项要素之一直接标记为 S4。

### 7.3 可能性（Likelihood）评分体系

| 等级 | 分值 | 标准 | 场景示例 |
|------|------|------|----------|
| L1 | 1 | 极低 | 高标准新建建筑，消防管理严格，历史无火灾 |
| L2 | 2 | 较低 | 一般商业建筑，有基本消防管理 |
| L3 | 3 | 中等 | 老旧建筑，多业态混合，管理一般，周边有隐患源 |
| L4 | 4 | 较高 | 易燃易爆场所，劳动密集型企业，历史有火警，管理混乱 |

**数据来源**：
- 场所类型标签（易燃易爆场所默认 L≥3）
- 隐患数量与类型（电气+易燃物堆积→L+1）
- 建筑年代与消防管理状况（若可获取）

### 7.4 可检测性（Detectability）评分体系

| 等级 | 分值 | 标准 | 说明 |
|------|------|------|------|
| D1 | 1 | 极易发现 | 肉眼一眼可见（如灭火器明显缺失） |
| D2 | 2 | 容易发现 | 稍加注意可发现（如灭火器压力表在黄色区域） |
| D3 | 3 | 需要专业知识 | 需要消防知识才能判断（如防火分区设置是否合规） |
| D4 | 4 | 极难发现 | 即使专业人员也需要仪器检测（如线路过载、隐蔽工程缺陷） |

**注意**：D值越高 = 越难发现 = 越危险（因为隐患可能长期存在不被察觉）。

### 7.5 综合风险等级

| 风险等级 | 风险值范围 | 颜色 | 含义 | 处置要求 |
|----------|-----------|------|------|----------|
| **I 级（低风险）** | 1-8 | 蓝 | 可接受风险 | 记录在案，下次巡检复查 |
| **II 级（一般风险）** | 9-18 | 黄 | 需要关注 | 限期整改（7-30天） |
| **III 级（较大风险）** | 19-36 | 橙 | 较高风险 | 限期整改（3-7天），通报安全责任人 |
| **IV 级（重大风险）** | 37-64 | 红 | 不可接受风险 | 立即整改，必要时停业整顿，上报消防部门 |

### 7.6 权重计算与 ANP 方法（预留）

基于 MDPI Fire (2024) 中使用的 ANP（网络分析法），可建立更精确的二级指标权重：

```
综合严重性 S = Σ(二级指标i × 权重Wi)
```

**二级指标示例**：
- 影响人员数量（人数越多，严重性越高）
- 是否涉及儿童/老人等弱势群体（是→S+1）
- 是否位于地下/高层（是→S+1）
- 是否疏散路径上的瓶颈位置（是→S+1）

此部分作为 v2 迭代优化项，初期使用简化的一级评分即可。

## 8. 降级策略（Degradation Strategy）

### 8.1 降级层级

| 层级 | 描述 | 触发条件 | 行为 |
|------|------|----------|------|
| **L0 全功能** | DeepSeek Vision + 向量检索 + LLM 生成 | API Key 正常，网络通畅 | 完整的隐患识别→法规匹配→报告生成链路 |
| **L1 规则增强** | DeepSeek Vision + 规则关键词匹配 | 向量库不可用，但 Vision API 可用 | 视觉识别正常，法规匹配降级为规则关键词 |
| **L2 离线本地** | 本地轻量模型 + 规则引擎 | Vision API 不可用 | 使用本地 YOLO 检测特定目标（灭火器/消火栓）+ 规则分类 |
| **L3 手动辅助** | 仅输出检查清单 | 所有 AI 能力不可用 | 按场所类型输出结构化消防检查清单，用户手动逐项检查打钩 |

### 8.2 各层级能力对比

| 能力 | L0 全功能 | L1 规则增强 | L2 离线本地 | L3 手动辅助 |
|------|-----------|------------|------------|------------|
| 视觉隐患识别 | ✅ LLM 全类别 | ✅ LLM 全类别 | ⚠️ 仅限训练过的目标 | ❌ |
| 隐患分类 | ✅ 全部分类树 | ✅ 全部分类树 | ⚠️ 仅目标类别 | ❌ |
| 风险量化 | ✅ 3D矩阵+场景感知 | ✅ 3D矩阵 | ⚠️ 简化2D矩阵 | ❌ |
| 法规匹配 | ✅ 语义向量+LLM重排 | ⚠️ 规则关键词 | ⚠️ 规则关键词 | ⚠️ 静态清单 |
| 整改建议 | ✅ LLM生成，场景化 | ✅ LLM生成 | ❌ 预设模板 | ❌ |
| 离线可用 | ❌ | ❌ | ✅ | ✅ |
| 响应速度 | 3-8秒 | 3-6秒 | <2秒 | 即时 |

### 8.3 降级决策逻辑

```text
function decideLevel():
    if (!API_KEY_AVAILABLE):
        if (LOCAL_MODEL_AVAILABLE): return L2
        else: return L3
    
    if (VECTOR_DB_HEALTH_CHECK_FAILS): return L1
    
    return L0
```

## 10. 技术栈与底座选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **多模态视觉识别** | DeepSeek Vision VL2 | 已在项目中使用，中文场景理解强，MoE架构性价比高 |
| **轻量火焰检测** | YOLOv8s-Nano（本地） | 可离线部署，火焰/烟雾快速预警，<2秒响应 |
| **法规语义匹配** | mxbai-embed-large (1024维) | 文献验证：监管文本语义匹配最优 |
| **向量数据库** | ChromaDB / LanceDB（嵌入式） | 轻量级，无需独立部署，适合移动端/小程序场景 |
| **法规文本处理** | 按条款边界分割 + 元数据标注 | 保持法规条款完整性，支持精确引用 |
| **LLM 生成** | DeepSeek LLM（文本生成） | 报告生成、整改建议、法规解释 |
| **离线降级** | YOLOv8s + 本地规则引擎 | 无网络时仍可提供基础检查能力 |

## 12. 验证清单

部署或改算法后建议确认：

1. 拍摄含明显隐患（如灭火器缺失 + 通道堵塞）的照片，所有隐患均被检出
2. 拍摄完全合规的场所照片，误报率 < 10%
3. 同一张照片重复提交，5分钟内 `meta.cacheHit === true`
4. 切换到不同场所类型（商场/工厂/学校），隐患分类结果随场所类型调整
5. 每条隐患标注的法规条款编号可在 GB 原文中找到对应内容
6. 风险等级为 IV 级（红色）的隐患在报告中高亮显示并带「立即整改」标识
7. 关闭 API Key 后，系统降级为 L1（规则增强）或 L2（离线本地），不白屏
8. 人工复核后，反馈数据正确写入 inspection_records，下次同类型场景识别准确率有所提升
9. 火焰/烟雾检测延迟 < 2 秒（独立快速通道）
10. 完整识别链路（拍照→报告）< 8 秒（L0 全功能模式）
